The Access Group module requires a complete backend and frontend implementation. First, the system 
must support the logical grouping of physical door controllers via a dedicated API endpoint. Second, a UI 
dashboard must be developed to allow administrators to drag and drop 'Personnel' into these groups for 
batch permission assignment. Finally, the system must implement a 'Time-Zone' filter, where certain 
Access Groups are only active during specific office hours (e.g., 09:00 to 18:00), requiring a background 
service to sync these permissions to the local door hardware. The Personnel module is the central 
identity store. Development must include a secure database schema that supports employee profiles 
with mandatory 10-digit ID verification logic. Furthermore, the system must integrate a file-upload 
service that accepts profile photographs, including automated image resizing and secure storage in an 
encrypted directory. Lastly, a search and filter interface must be built, allowing HR managers to query 
personnel by department, status, or ID number with sub-second latency. The Badge Holder feature 
manages the hardware-software handshake. The initial scope requires the implementation of a reader-
service that listens for physical RFID badge swipes and validates them against the Personnel database. 
The system must support a 'Strict Numeric' format for all Badge IDs. Additionally, a management 
console is required to 'Block/Unblock' badges instantly in case of loss. All badge transactions must be 
logged in a persistent audit trail for security compliance.
