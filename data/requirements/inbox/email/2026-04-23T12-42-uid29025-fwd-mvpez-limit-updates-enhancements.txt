With the following web.config values:

Please implement the logic as described below:

*General Rule:* All limits must not exceed both the service plan limit and
the value defined in web.config.

*Standard Badge (Max: 200):*

   - If the service plan value is set to 0 (unlimited), the maximum allowed
   should default to the web.config value (200).

*BLE Credential (Max: 200):*

   - The limit should be the lower value between the configuration and the
   service plan.

*NFC Credential (Max: 200):*

   - The limit should be the lower value between the configuration and the
   service plan.

*Doors (Max: 50):*

   - The limit should be the lower value between the configuration and the
   service plan.

*Badges per Person (Max: 4):*

   - This is the total number of badges a person can have (Standard, BLE,
   NFC, or any combination), capped at 4.

*Additional Enhancements:*

   1. *Allow Re-generation of BLE Credentials*
      - Add a button or link to allow users to regenerate a BLE credential
      badge.
      2.

   *Allow Deletion of a Person with All Associated Data* Provide an option
   to delete a person along with all related records:
   - Delete *Operators*
      - Remove OperatorId from *DevicePartitions *table
      - Delete badges from *BadgeAccess *table
      - Delete badges from *BadgeUpdate *table
      - Delete person from *Person *table
      - Remove PersonId from *DevicePartitions *table

Please let me know if you have any questions.
